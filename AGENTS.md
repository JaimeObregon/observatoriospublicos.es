# Instrucciones para agentes de IA

Lee @README.md para conocer la descripción general del proyecto y @package.json para conocer los scripts disponibles y otras configuraciones.

## Filosofía

- Simplicidad, mantenibilidad y código limpio.
- Evita complejidad técnica innecesaria.
- Stack mínimo y simple, sin bloat.
- Evita añadir nuevas dependencias.

## Formato de código

- Comillas simples (').
- Sin puntos y coma (semicolons).
- Sigue el formato de Prettier para todo lo demás.

## Estilo

- Módulos ES6 (import/export).
- `async`/`await` para asincronía.
- Programación funcional siempre que sea posible.
- Nombres de símbolos (funciones, variables, clases…) SIEMPRE en inglés.
- Nombres cortos, semánticos; sin abreviaturas cuando sea posible.
- camelCase para variables/funciones, PascalCase para clases.
- Comentarios y documentación SIEMPRE en castellano.
